# 回款数据处理规则

## 固定工作约束

- 程序启动后由用户选择 `家电` 或 `数码`，并输入对应的原始数据文件地址；不再依赖固定的 `data` 目录。
- Python 环境：本目录的 uv 项目环境，统一使用 `uv run python main.py` 运行。
- 处理代码：本项目目录中的 `main.py`。
- 本文件用于持续记录数据处理要求、输入输出约定和运行方法。
- 每次新增或调整处理要求时，必须同步更新 `main.py` 与本文件。
- 原始数据默认只读；处理结果应另存为新文件，除非要求明确允许覆盖。
- 每次运行必须先将用户选择的数据表转换为 `.xlsx` 工作副本。
- 所有转换副本和后续生成的新表统一存放在本项目的 `output` 目录。
- 后续清洗、筛选、计算、汇总等处理只能针对 `output` 中的工作副本，禁止修改用户提供的原表。

## 输入数据

- 家电原始数据示例：`2026年以旧换新补贴明细.xlsx`。
- 数码原始数据示例：`2026年数码补贴明细(1).xlsx`。
- 当前支持 `.xlsx`、`.xls`、`.xlsm`、`.csv` 和 `.tsv` 数据表；`.xlsm` 转为 `.xlsx` 时不保留 VBA 宏。
- `.xlsx` 原表会原样复制到 `output`，其他支持的格式会转换后写入 `output`。
- 文件不存在或格式不受支持时程序会报错，避免静默遗漏数据。

## 家电明细表统一结构

- 所有家电明细 Sheet 的表头统一为：`拨付批次`、`交易时间`、`交易参考号`、`交易订单号`、`销售企业`、`商户编号`、`销售金额`、`其他支付`、`实收销售金额`、`补贴金额`、`补贴比例`、`SN码`、`所在地区`、`商品编码`、`能耗等级`、`编码品类`、`商品名称`、`发票金额`、`发票号`、`ID`。
- `交易完成时间` 统一改为 `交易时间`。
- `销方名称`、`销售企业名称` 统一改为 `销售企业`。
- `核销商编` 统一改为 `商户编号`。
- `应收销售金额` 统一改为 `销售金额`。
- `销售金额` 固定为第 7 列，`其他支付` 固定为第 8 列。
- 原本没有 `发票金额` 的明细 Sheet，在 `发票号` 前增加空白的 `发票金额` 列；不推断或填造金额。
- `发票金额`、`发票号`、`ID` 依次固定为第 18、19、20 列。
- 统一结构时仅保留实际连续数据区域并移除原表扩展至 Excel 最大行列的无效空白格式；输出工作副本以数据处理为目的，不继承原表单元格样式。

## 数码明细表统一结构

- 数码明细统一为19列：`拨付批次`、`交易时间`、`交易参考号`、`交易订单号`、`销售企业`、`商户编号`、`销售金额`、`其他支付`、`实收销售金额`、`补贴金额`、`编码品类`、`商品名称`、`SN码`、`IMEI1码`、`IMEI2码`、`商品编码`、`所在地区`、`发票号`、`备注`。
- `参考号`、`订单号`、`经销商编号`、`补贴销售金额`、`品类`、`商品明细`、`发票号码`分别统一为`交易参考号`、`交易订单号`、`商户编号`、`补贴金额`、`编码品类`、`商品名称`、`发票号`。
- 保留数码数据特有的 `IMEI1码`、`IMEI2码` 和 `备注`，不套用家电20列结构。

## 明细整合与筛选

- 保留 `汇总` Sheet 名称，但清空原有内容后按当前整合明细重新生成汇总。
- 将 `汇总` 之外的全部明细 Sheet 按所选数据类型的标准结构纵向合并为一个 `整合明细` Sheet；以后新增的明细 Sheet 自动参与合并。
- 家电仅保留 `商户编号` 精确等于 `89813015722APT1` 的记录；数码仅保留 `商户编号` 精确等于 `89813014812B06R` 的记录。
- 合并时保留符合条件的全部原始记录，不执行去重。
- 家电结果保存为 `output\家电回款明细.xlsx`；数码结果保存为 `output\数码回款明细.xlsx`。
- 转换阶段产生的中间工作副本在处理成功后移除。

## `整合明细` 格式

- 运行时检测系统字体：已安装 `Maple Mono NF CN` 时使用该字体，否则使用 `微软雅黑`。
- 标题行与数据区字号统一为11号。
- 所有行高统一为20。
- 各列使用实际字体文件逐单元格测量11号字体的像素宽度，再换算为 Excel 列宽，并额外增加25%的渲染安全余量和单元格边距，确保内容完整显示。
- 冻结标题行，即冻结窗格位置为 `A2`。
- 标题行使用黑色填充、白色字体。
- 标题行全部设置为水平、垂直居中。

## 汇总表

- 以 `财务大类` 和 `品牌` 为分组依据，对 `补贴金额` 同时进行求和与带符号计数。
- `汇总` Sheet 的字段依次为：`财务大类`、`品牌`、`补贴金额合计`、`补贴金额计数`。
- `补贴金额合计` 使用十进制累加并按四舍五入保留两位小数。
- `补贴金额` 小于0的记录计数为 `-1`，其他非空补贴金额记录计数为 `+1`；分组计数为这些计数值之和。
- 在 `汇总` Sheet 底部增加 `合计` 行，统计全部分组的补贴金额合计与带符号计数合计，并使用粗体显示。
- `汇总` Sheet 中连续且相同的 `财务大类` 单元格纵向合并，并设置为水平、垂直居中；标题行和底部合计行不参与合并。
- `汇总` Sheet 应用与 `整合明细` 相同的字体、字号、行高、自动列宽、标题样式和冻结标题行设置。

## 新增字段

- 在 `整合明细` 末尾依次增加 `财务大类`、`品牌` 两列。
- 家电 `财务大类` 映射：`A04-空调` → `空调`、`A06-热水器` → `厨卫`、`A01-电视机` → `电视`、`A03-洗衣机` → `洗衣机`、`A02-电冰箱` → `冰箱`。
- 数码 `财务大类` 映射：`B01-手机` → `手机`、`B02-平板` → `平板`、`B03-智能手表手环` → `智能穿戴`。
- 若出现映射表未配置的 `编码品类`，程序报错，避免生成错误分类。
- `品牌` 从 `商品名称` 中按关键词提取，匹配时不区分英文字母大小写，并支持品牌出现在商品名称开头、中间或末尾。
- 家电品牌词典覆盖：`A.O.史密斯`、`卡萨帝`、`小天鹅`、`西门子`、`奥克斯`、`万家乐`、`Leader`、`COLMO`、`TCL`、`小鸭`、`美菱`、`晶弘`、`华凌`、`格力`、`海尔`、`美的`、`海信`、`创维`、`科龙`、`博世`、`统帅`。
- 数码品牌词典覆盖：`华为`、`小米`、`荣耀`、`OPPO`、`vivo`、`一加`、`苹果`、`三星`、`小天才`、`作业帮`、`学而思`、`洪恩`、`步步高`；其中 `REDMI` 归入 `小米`、`iQOO` 归入 `vivo`。
- 品牌提取后统一归并：`COLMO` → `美的`、`卡萨帝` → `海尔`、`Leader` → `海尔`、`科龙` → `海信`、`统帅` → `海尔`、`华凌` → `美的`、`晶弘` → `格力`。
- 同表中已确认裸型号 `TNDD20-08AIDE` 属于 `小天鹅`，按型号补充识别。
- 经业务确认，裸型号 `65A6Q` 属于 `海信`，按型号补充识别。
- 商品名称无法可靠识别品牌时，`品牌` 保留为空，并在运行日志中报告未识别数量，不凭型号猜测。
- 直接提取后仍为空时，程序从同一 `整合明细` 中已识别品牌的商品名称学习“型号标识 → 品牌”关系；仅当相同型号标识在已有数据中唯一对应一个品牌时自动回填。
- 同一型号标识对应多个品牌时视为有歧义，不自动回填；运行日志分别报告推断成功数和最终未识别数。

## 当前运行方法

```powershell
uv run python main.py
```

运行后按提示选择数据类型并输入文件地址：

```text
请选择数据类型：
1. 家电
2. 数码
请输入序号：
请输入原始数据文件地址：
```

文件地址支持普通 Windows 路径、带引号的拖入路径和 `file:///C:/...` 地址。

# 回款数据处理工具

本项目用于处理家电和数码补贴明细。程序会将原始数据转换为 `.xlsx` 工作副本，统一明细结构，筛选指定商户，补充财务大类与品牌，并生成整合明细和汇总表。

## 环境要求

- Python 3.14 或更高版本
- [uv](https://docs.astral.sh/uv/)
- 项目依赖由 `pyproject.toml` 和 `uv.lock` 管理

## 快速开始

在项目目录中运行：

```powershell
uv run python main.py
```

根据提示选择数据类型、输入要筛选的商户编号并提供原始文件地址：

```text
请选择数据类型：
1. 家电
2. 数码
请输入序号：
请输入要筛选的商户编号：
请输入原始数据文件地址：
```

文件地址支持以下形式：

- 普通 Windows 路径
- 带引号的拖入路径
- `file:///C:/...` 地址

## 输入与输出

### 支持的输入格式

程序支持 `.xlsx`、`.xls`、`.xlsm`、`.csv` 和 `.tsv` 文件。

- `.xlsx` 文件会原样复制为工作副本。
- 其他受支持的格式会转换为 `.xlsx`。
- `.xlsm` 转换后不保留 VBA 宏。
- 文件不存在或格式不受支持时，程序会直接报错。

输入文件示例：

- 家电：`2026年以旧换新补贴明细.xlsx`
- 数码：`2026年数码补贴明细(1).xlsx`

### 输出文件

所有工作文件和结果均存放在 `output` 目录：

- 家电：`output\家电回款明细.xlsx`
- 数码：`output\数码回款明细.xlsx`

处理成功后，转换阶段产生的中间工作副本会被移除。

## 处理流程

1. 用户选择数据类型，输入要筛选的商户编号并提供原始文件。
2. 程序在 `output` 目录中创建 `.xlsx` 工作副本。
3. 各明细 Sheet 按对应数据类型统一表头和列顺序。
4. 程序筛选指定商户，并将全部有效明细纵向合并至 `整合明细` Sheet。
5. 程序补充 `财务大类` 和 `品牌` 字段。
6. 程序清空并重建 `汇总` Sheet。
7. 程序应用统一格式并写入最终结果文件。

## 明细处理规则

### 通用规则

- `汇总` 之外的全部 Sheet 均视为明细表；以后新增的明细 Sheet 会自动参与合并。
- 合并时保留符合条件的全部原始记录，不执行去重。
- 原 `汇总` Sheet 仅保留名称，内容会按本次整合结果重新生成。
- 统一结构时仅保留实际连续数据区域，移除扩展至 Excel 最大行列的无效空白格式。
- 工作副本用于数据处理，不继承原表单元格样式。

### 家电明细

程序提示用户输入要筛选的商户编号，仅保留 `商户编号` 与输入值完全一致的
记录。商户编号不能为空，不自动使用默认值。

标准结构共 20 列：

1. `拨付批次`
2. `交易时间`
3. `交易参考号`
4. `交易订单号`
5. `销售企业`
6. `商户编号`
7. `销售金额`
8. `其他支付`
9. `实收销售金额`
10. `补贴金额`
11. `补贴比例`
12. `SN码`
13. `所在地区`
14. `商品编码`
15. `能耗等级`
16. `编码品类`
17. `商品名称`
18. `发票金额`
19. `发票号`
20. `ID`

表头统一规则：

- `交易完成时间` → `交易时间`
- `销方名称`、`销售企业名称` → `销售企业`
- `核销商编` → `商户编号`
- `应收销售金额` → `销售金额`

原表缺少 `发票金额` 时，在 `发票号` 前增加空白列，不推断或填造金额。

### 品牌识别

品牌从 `商品名称` 中按关键词提取，匹配英文字母时不区分大小写，关键词可位于商品名称的任意位置。

家电品牌词典：

`A.O.史密斯`、`卡萨帝`、`小天鹅`、`西门子`、`奥克斯`、`万家乐`、`Leader`、`COLMO`、`TCL`、`小鸭`、`美菱`、`晶弘`、`华凌`、`格力`、`海尔`、`美的`、`海信`、`创维`、`科龙`、`博世`、`统帅`。

`华为`、`小米`、`荣耀`、`OPPO`、`vivo`、`一加`、`苹果`、`三星`、`小天才`、`作业帮`、`学而思`、`洪恩`、`步步高`。

别名与子品牌归并规则：

| 识别结果 | 归并品牌 |
| --- | --- |
| `COLMO` | `美的` |
| `华凌` | `美的` |
| `卡萨帝` | `海尔` |
| `Leader` | `海尔` |
| `统帅` | `海尔` |
| `科龙` | `海信` |
| `晶弘` | `格力` |
| `REDMI` | `小米` |
| `iQOO` | `vivo` |

已确认的裸型号映射：

- `TNDD20-08AIDE` → `小天鹅`
- `65A6Q` → `海信`

直接识别失败后，程序会从同一 `整合明细` 中已识别的数据学习“型号标识 → 品牌”关系：

- 相同型号仅对应一个品牌时，自动回填。
- 相同型号对应多个品牌时，视为存在歧义，不自动回填。
- 仍无法可靠识别时，品牌保留为空，不凭型号猜测。
- 运行日志会报告推断成功数和最终未识别数。

## 汇总表规则

- 按 `财务大类` 和 `品牌` 分组。
- 字段顺序为 `财务大类`、`品牌`、`补贴金额合计`、`补贴金额计数`。
- `补贴金额合计` 使用十进制累加，并按四舍五入保留两位小数。
- 负数补贴记录计为 `-1`，其他非空补贴记录计为 `+1`，分组计数为带符号计数之和。
- 表格底部增加 `合计` 行，汇总补贴金额和带符号计数，并使用粗体。
- 连续且相同的 `财务大类` 单元格纵向合并；标题行和合计行不参与合并。

## 工作表格式

`整合明细` 和 `汇总` 使用相同的基础格式：

- 优先使用系统字体 `Maple Mono NF CN`，未安装时使用 `微软雅黑`。
- 标题行和数据区字号均为 11 号。
- 所有行高均为 20。
- 标题行使用黑色填充、白色字体，并水平、垂直居中。
- 冻结标题行，冻结窗格位置为 `A2`。
- 按实际字体文件逐单元格测量 11 号字体的像素宽度，换算为 Excel 列宽后增加 25% 的渲染安全余量和单元格边距。

## 项目维护约束

- 原始数据只读，禁止覆盖用户提供的文件。
- 清洗、筛选、计算和汇总只能针对 `output` 中的工作副本。
- 处理逻辑统一维护在 `main.py` 中。
- 每次新增或调整业务规则时，必须同步更新 `main.py` 和本文件。
- 项目统一使用 `uv run python main.py` 运行。
